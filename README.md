# scrum-notes
**SCRUM:**
Scrum is a framework used to manage and complete complex project.

It follows agile method..

SCRUM ROLES -- 

1.Product owner(po)
                 *decides what to build.
                 *manages product Backlogs.
                 *Sets Priorities.
                2.Scrum Master
                 removes blockers.
                 *supports team.
                3.Development Team
                  *Developers
                  *Designers
                  *Testers
                  *One who builds product.
**SPRINT:**
A Sprint is a short time-boxed period (1â€“2 weeks)  
where the team completes selected tasks from the Product Backlog.

**Scrum Events:**
1.Sprint Planning -Daily Scrum (Stand-up)
                  15-minute daily meeting:
                  - What did I do yesterday?
                  - What will I do today?
                  - Any blockers?
2.Sprint Review- Team demonstrates completed work.
3.Sprint Retrospective-Team discusses:
                          - What went well
                          - What can improve
                          - Action items for next sprint
4.Product Backlog-A prioritized list of all features, improvements, and fixes.

**User Story:**
 A User Story describes a feature from the user's perspective.

 EX: As a <user>,  
I want <feature>,  
So that <benefit>.


**Triage:**
Triage is the process of reviewing and prioritizing issues or bugs based on severity and impact.

It helps the team decide:
- What to fix first
- What can wait
- What is minor

**Bug Triage Process:**

When multiple bugs are reported:
1. Review the issue
2. Check impact
3. Assign severity
4. Set priority
5. Add to sprint if required
